# 结构体
## 概念
- Golang没有class类，只有struct结构体
- 不是纯粹的面向对象
## 定义
```Golang
type Person struct {
    Name string
    Age int
    School string
}
```
## 对象实现
```Golang
方式一
var p1 Person
fmt.Println(p1)
p1.Name = "Rapeter"
p1.Age = 18
p1.School = "Harvard"
fmt.Println(p1)

方式二
var s1 Student = Student{"Tom",20,"Stanford"}
fmt.Println(s1)

方式三
var s2 Student = Student{
        Name: "Jerry",
        Age: 21,
    }
fmt.Println(s2)

//返回结构体指针
var s *Student = new(Student)
//s 为指向地址
(*s).Name = "Jane"
(*s).Age = 25
(*s).School = "MIT"
fmt.Println(*s)
s.Name = "Alice"//实际转化成（*s）.Name = "Alice"

//返回结构体的指针类型
var s3 *stu = &stu{"oh god",24}
    fmt.Println(*s3)

方式四
var s0 *Student = &Student{"Bob",22,"Caltech"}
```
## 结构体转换
- 进行转换时需要有相同字段
```Golang
type Person struct {
    Name string
    Age int
}
type Student struct {
    Name string
    Age int
}
//方式一
    var s Student = Student{"Tom", 20}
    var p Person = Person{"Jerry", 25}
    s = Student(p)//结构体转换，要用强制转换
    fmt.Println(s)
```

```Golang
type Student struct {
    Name string
    Age int
}
type Stu Student//Stu等价于Student结构体
//方式二
    var stu Stu = Stu{"Tom", 20}
    fmt.Println(stu)
    s = Student(stu)
```
# 方法
## 值传递与值类型方法
- 在方法调用中,是值拷贝传递
```Golang
type Person struct {
    Name string
    Age int
}
func (p Person) test() {
    fmt.Println(p.Name, p.Age)
}
var p Person = Person{"Tom", 20}
    p.test()//是一个值传递，调用的是值类型的方法，不改变原本值
    (&p).test()//是一个指针传递，依旧可以调用方法，该形式虽然是指针传递，但是本质上还是值传递，不能改变原本值
```
## 指针传递与指针类型方法
```Golang
type Person struct {
    Name string
    Age int
}
func (p *Person) test2() {
    p.Name = "Jerry"
    p.Age = 25
    fmt.Println(p.Name, p.Age)
}

p.test2()//是一个指针传递，调用的是指针类型的方法，改变原本值
(&p).test2()//是一个指针传递，调用的是指针类型的方法，改变原本值
```
## 基础数据类型方法
- 方法绑定在指定的数据类型上，而不是在结构体上
- 基本数据类型也可以有方法
```Golang
type integer int
func (i integer) test3() {
    i = 20
    fmt.Println(i)
}
var i integer = 10
i.test3()//调用的是值类型的方法，不改变原本值
```
## 指针绑定
- 方法访问控制权限，跟函数一样,方法首字母小写，只能在本包访问，大写则可被其他包访问
```Golang
type Student struct {
    Name string
    Age int
}
func (s *Student) String() string{
    str := fmt.Sprintf("Name: %s, Age: %d", s.Name, s.Age)//拼成一个字符串
    return str
}
stu := Student{"Mike", 25}
    fmt.Println(stu)// 输出-->{Mike 25}
    fmt.Println(&stu)// 输出-->Name: Mike, Age: 25
```
- 传入地址才和指针绑定，绑定了String方法，会自动调用
## 方法与函数的区别
- 前者需要绑定数据类型
- 后者不需要绑定数据类型
- 方法调用需要传递接收者，函数调用不需要
例:函数名(参数)  变量.方法名(参数)
- 函数指定什么类型，传入参数就是什么类型
- 方法不一样，接收参数为值类型，可以传入指针类型
- 接收参数为指针类型，可以传入值类型

# 封装
## 概念
- 封装：就是把抽象的字段和对字段的操作封装在一起，对外提供一个接口，隐藏内部的实现细节。
- 优点：
1. 隐藏实现细节，提高代码的可读性和可维护性。
2. 提高代码的复用性。
## 实现的几种方法
1. 将结构体，属性的首字母小写
2. 给结构体所在包提供一个工厂模式的函数，首字母大写
3. 提供一个首字母大写的Set方法，用对应属性并赋值
4. 提供一个首字母大写的Get方法，返回对应属性的值
## 代码实例
- main包
```Golang
package main
import "fmt"
import "gocode/project01/unit10/demo6/model"
func main() {
//代码实现:
    p := model.NewPerson("Tom", 20)
    p.SetName("Jerry")
    fmt.Println(p.GetName())
    fmt.Println(*p)
    fmt.Println(p.Name)
    //都可以访问
}
```
- model包
```Golang
package model
type person struct {
    Name string
    Age  int
}
//工厂模式
func NewPerson(name string, age int) *person {
    return &person{name,age}
}
//定义set与get方法，对属性进行封装
func (p *person) SetName(name string) {
    p.Name = name
    fmt.Println("set name success")
}
func (p *person) GetName() int {
    return p.Name
}
```
# 继承
- 优点：代码复用，以及代码拓展
## 代码实例
==以下的代码实例都以Animal和Cat结构体为示例==
- 定义父类与子类
```Golang
//定义父类
type Animal struct {
    Weight int
    Age int
}
func (an *Animal) Shout(){
    fmt.Println("Animal Shout")
}
func (an *Animal) Show(){
    fmt.Printf("Age : %d, Weight : %d\n", an.Age, an.Weight)
}
//定义子类
type Cat struct {
    //加入匿名结构体
    Animal
    Age int//重复属性
}
//重复属性调用
func (c *Cat) Show(){
    fmt.Printf("····Cat Age : %d, Weight : %d\n", c.Age, c.Weight)
}
func (c *Cat) Scratch(){
    fmt.Println("Cat Scratch")
}
```
- 调用
```Golang
func main() {
    cat := &Cat{}
    cat.Animal.Weight = 5
    cat.Age = 2
    ↓↓ ↓↓ ↓↓
    //等同于
    //cat.Weight = 5
    //cat.Animal.Age = 2
    
    cat.Show()
    cat.Shout()
    cat.Scratch()
}
```
## 注意事项一
- 结构体可以使用嵌套匿名结构体的所有属性与方法
例:Cat结构体可以调用Animal结构体的方法
- 且匿名结构体字段访问可以简化
```Golang
cat.Animal.Weight = 5
↓ ↓ ↓ ↓ ↓ ↓
cat.Weight = 5
```
- 当结构体与匿名结构体有相同的属性与方法时，采用就近访问原则
- 就近访问原则：
1. 重名属性会赋值给cat结构体里的Age属性,而不是Animal结构体里的Age属性。
2.  重名方法也会先调用cat结构体的Show方法，类似重载。
- 解决方法
可以通过匿名结构体名区分
```Golang
cat.Animal.Age = 12//会赋值给Animal结构体里的Age属性
    cat.Animal.Show()//调用Animal结构体的Show方法
    cat.Show()//调用cat结构体的Show方法
```
## 注意事项二
- 多重继承：一个结构体中嵌套了多个匿名结构体，就可以实现多重继承，==尽量不使用多重继承==
- 嵌入的匿名结构体中有相同属性与方法时，可以通过匿名结构体名进行区分
- 代码示例
```Golang
type A struct {
    a int
    b string
}
type B struct {
    c int
    d string
    a int//重复属性
}
type C struct {
    A
    B
    //基本数据类型
    int
}
//赋值
c := C{A{10,"aaa"},B{12,"bbb",23},888}
fmt.Println(c.b)
fmt.Println(c.d)
//fmt.Println(c.a)//出现了ambiguous selector错误，因为c.a是A的属性，B也有a属性，无法区分
 fmt.Println(c.A.a)
 fmt.Println(c.B.a)
```
- 结构体的属性可以是基本数据类型
```Golang
type C struct {
    //基本数据类型
    int
}
fmt.Println(c.int)
```
- 基础赋值
```Golang
c := C{A{10,"aaa"},B{12,"bbb",23},888}
```
- 指定赋值
```Golang
cc := C{
        A{
            a : 10,
            b : "aaa"},
        B{
            c : 12,
            d : "bbb",
            a : 23},
        888}
    fmt.Println(cc)
```
- 指针赋值与调用
```Golang
type D struct {
    *A
    *B
    //基本数据类型
    int
}
//嵌入匿名结构体的指针也可以
    c1 := D{&A{10,"aaa"},&B{12,"bbb",23},888}
    fmt.Println(c1)//会获取A和B的地址
    fmt.Println(*c1.A)
    fmt.Println(*c1.B)//因为A B都以指针形式嵌套，所以用实例的指针获取A，B的属性
```
- 结构体的属性也可以是结构体
```Golang
type A struct {
    a int
    b string
}
type E struct {
    a int
    b string
    c A //组合模式
}
e := E{10,"eee",A{11,"aaa"}}
    fmt.Println(e)//嵌套嵌套嵌套
    fmt.Println(e.c.a)
```
# 接口
## 概念
- 接口（interface）是一种​**​抽象类型​**​，用于定义对象的行为规范而非具体实现
- 接口通过 `type`和 `interface`关键字声明，包含一组​**​方法签名​**​（方法名、参数和返回值）
- 接口中可以定义一组方法，但不需要实现，不需要方法体，且接口中不包含任何变量
- 实现接口要实现所有的方法才是实现接口
- 接口的目的是为了规范定义，具体由别人实现即可
- Golang中的接口，不需要显式的实现接口且没有implement关键字
## 代码示例
```Golang
type SayHello interface {//interface关键字定义接口
    //声明没有实现的方法
    Say()
}
//实现接口 -> 定义一个结构体
type Chinese struct {
}//将方法绑定到结构体上，进行实现

func (c Chinese) Say() {//Say是方法需要传参，所以需要加括号
    fmt.Println("你好")
}
//定义一个函数，专门用来测试接口的使用,用来接收具备Say()接口能力的变量
func greet(s SayHello) {//接收接口变量
    s.Say()
}
//调用
greet(c)
↓ ↓ ↓ ↓
c.Say()
```
## 注意事项
- 接口本身不能创建实例
```Golang
var s SayHello
s.Say() //报错，接口没有实现方法
```
- 但可以指向一个实现该接口的变量
```Golang
type SayHello interface {//interface关键字定义接口
    //声明没有实现的方法
    Say()
}
type Chinese struct {
}
//调用
c := Chinese{}
var s SayHello = c
s.Say()//该方法可以调用，因为c实现了Say()方法
```
## 自定义数据类型
- 只要是自定义数据类型就可以实现接口，不一定非得是结构体
```Golang
//自定义数据类型
type intType int
func (i intType) Say() {
    fmt.Println("intType")
}

//调用
var i intType = 10
    var s1 SayHello = i
    i.Say()//调用自定义数据类型的方法
	↓ ↓ ↓ ↓ ↓ ↓
    s1.Say()//调用自定义数据类型的方法
```
## 注意事项
- 一个自定义类型可以实现多个接口
```Golang
type AInterface interface {//多个接口实现
    a()
}
type BInterface interface {
    b()
}
type Stu struct {
}
func (s Stu) a() {
    fmt.Println("a")
}
func (s Stu) b() {
    fmt.Println("b")
}
//调用
    var s Stu
    s.a()
    s.b()
    var a AInterface = s
    var b BInterface = s
    //a.a() = s.a()
    //b.b() = s.b()
    a.a()
    b.b()
```
- 一个接口可以继承多个接口，但是要实现该接口，需要实现所有接口的方法
```Golang
type AInterface interface {//多个接口实现
    a()
}
type BInterface interface {
    b()
}
type CInterface interface {
    AInterface
    BInterface
    c()
}
type C struct {
}
func (c C) c() {
    fmt.Println("c")
}
func (c C) a() {
    fmt.Println("c-a")
}
func (c C) b() {
    fmt.Println("c-b")
}
//调用
    var c C
    var CI CInterface = c
    CI.a()
    CI.b()
    CI.c()
```
- interface类型默认是一个指针，若没有对interface初始化就使用，会输出nil(空)
- 空接口没有任何方法
- 用途:可以用该方法定义接收任何类型的形参，做类似泛型的效果
```Golang
var s Stu
type E interface {
}
//调用
    var e E = s
    fmt.Println(e)
    var e2 interface{} = s
    fmt.Println(e2)
```
# 多态
## 概念
- 多态通过接口实现，可以按照统一的接口调用不同的实现，此时接口变量就呈现不同的形态
## 代码示例
```Golang
//接口定义
type SayHello interface {//interface关键字定义接口
    //声明没有实现的方法
    Say()
}
//实现接口 -> 定义一个结构体
type Chinese struct {
    name string
}
type British struct {
    name string
}//将方法绑定到结构体上，进行实现

func (c Chinese) Say() {//Say是方法需要传参，所以需要加括号
    fmt.Println(c.name + "你好")
}
func (b British) Say() {
    fmt.Println(b.name + "Hello")
}

//定义一个函数，专门用来测试接口的使用,用来接收具备Say()接口能力的变量
func greet(s SayHello) {//接收接口变量  //s SayHello -> 多态的参数
    s.Say()
}//调用接口方法，体现多态

//调用
	c := Chinese{}
    b := British{}
    greet(c)
    greet(b)
```
- 多态数组
```Golang
	var arr [3]SayHello
    arr[0] = Chinese{"琳"}
    arr[1] = British{"Lily"}
    arr[2] = Chinese{"盖"}
    fmt.Println(arr)
```
# 断言
## 概念
- 可以直接判断是否是该类型的变量
- 例：value，ok = element.(T),value是变量的值，ok是一个bool类，element是interface变量，T是断言的类型
- 只能调用共有方法
- 用断言判断s的类型，再调用对应的方法
## 不同语法
```Golang
//定义两个结构体
package main
import "fmt"
type SayHello interface {
    Say()
}
type Chinese struct {
    name string
}
type British struct {
    name string
}
func (c Chinese) Say() {
    fmt.Println(c.name + "你好")
}
func (c Chinese) Conly() {
    fmt.Println("中国人独有")
}
func (b British) Say() {
    fmt.Println(b.name + "Hello")
}
func (b British) Bonly() {
    fmt.Println("英国人独有")
}
//定义接口的方法
func greet (s SayHello) {
    s.Say()
    //判断s的类型
}

//定义断言
第一种形式
    var ch Chinese = s.(Chinese)//s是否能转成chinese类型
    ch.Conly()
第二种形式
    ch2,flag := s.(Chinese)
    if flag == true {
        ch2.Conly()
    }else{
        fmt.Println("不是中国人")
    }//解决不同类型的返回值
第三种形式
    if ch,flag := s.(Chinese);flag == true{
        ch.Conly()
    }else{
        fmt.Println("not Chinese")
    }

//switch type的用法
	switch s.(type){//type属于go中的一个关键字,固定写法
        case Chinese:
            ch := s.(Chinese)
            ch.Conly()
        case British:
            b := s.(British)
            b.Bonly()
    }
```
